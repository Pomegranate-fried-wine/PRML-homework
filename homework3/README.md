
# 实验报告：基于LSTM的空气质量预测  （具体实验报告请看PDF文件）

## 1. 数据集与问题描述  
本实验使用了来自北京美国大使馆的**空气质量数据集**，该数据集包含2010年至2014年五年内逐小时的空气污染（PM2.5）浓度和气象数据【21†L148-L156】【36†L47-L55】。原始特征包括日期时间、PM2.5浓度、露点、温度、气压、风向、风速、累计降雪小时和累计降雨小时等（对应UCI库中的年、月、日、小时、pm2.5、DEWP、TEMP、PRES、cbwd、Iws、Is、Ir）【21†L158-L164】【36†L86-L94】。我们的目标是利用前序时刻的污染值和气象特征，通过堆叠LSTM网络预测下一小时的PM2.5浓度。这个**多变量时间序列预测**问题需要模型学习各个气象变量间的关联及其对污染的影响，典型的做法是在LSTM中输入多维特征序列【42†L30-L38】【42†L125-L134】。  

## 2. 数据预处理  
1. **时间序列索引**：首先将日期列(`date`)转换为`datetime`类型，并设置为时间索引，保证数据的时序连续性【42†L83-L92】。  
2. **缺失值处理**：检查数据集中的缺失值情况，由于原始数据可能存在NA值，采用`fillna(0)`填充为0，避免模型输入出现空值。  
3. **风向编码**：风向(`wnd_dir`)是类别型变量，使用标签编码（LabelEncoder）将其转换为数值形式，便于神经网络处理。  
4. **归一化**：将所有特征使用`MinMaxScaler`缩放到[0,1]区间，使得不同量纲的特征具有可比性，有助于模型快速收敛【42†L109-L117】。实践中，将整个数据集按行拼接，`fit_transform`后得到缩放后的数组。  

## 3. 特征工程：序列构造  
为了让LSTM学习时序依赖，我们将多变量数据重构为监督学习格式（滑动窗口）【42†L125-L134】。具体地，定义`look_back=12`，表示用前12小时的数据预测第13小时的PM2.5浓度。通过遍历数据集构造样本：每个样本的输入`X[i]`为第`i`时刻前12个时刻的所有特征值，输出`y[i]`为第`i`时刻的污染值【42†L125-L134】。经此转换后，数据维度为`(样本数, 12, 特征数)`，同时构造相应的一维目标序列。此方式类似GFG示例中将时间序列转换为监督学习序列的步骤【42†L125-L134】。  

## 4. 模型架构与训练  
1. **数据集划分**：将构造好的序列数据按80/20比例划分为训练集和测试集，其中前80%作为训练样本，后20%用于评估模型泛化性能。  
2. **网络结构**：采用Keras构建堆叠式LSTM网络。模型由两层LSTM单元组成：第一层有64个单元且`return_sequences=True`（返回序列输出），紧接一个Dropout层（比例0.2）防止过拟合；第二层LSTM有32个单元，后接另一个Dropout层（0.2）。最后接一层全连接(Dense)输出单个值【42†L159-L168】。这种**双层LSTM+Dropout**的结构能够提取长期时序特征并抑制过拟合，符合多维时间序列预测的常见做法【42†L159-L168】。  
3. **损失函数与优化器**：模型使用**平均绝对误差(MAE)**作为损失函数，优化器采用Adam。在训练过程中监控验证集的MAE。  
4. **早停策略**：引入EarlyStopping回调，当验证损失在连续5个周期内无改善时停止训练，并恢复最优权重，以避免过拟合【44†L209-L214】。早停能够自动截断冗余训练步骤，通常能提高模型的泛化能力【44†L209-L214】。
5. **训练过程**：在训练集上运行最多20个epoch，批量大小为64，不打乱时序（`shuffle=False`）。训练过程中记录训练损失和验证损失（MAE）曲线，用于后续分析。  

## 5. 预测与逆归一化  
训练完成后，在测试集上进行预测，得到标准化尺度下的预测值`y_pred`。由于我们对所有特征均进行了`MinMaxScaler`缩放，预测结果需要逆变换到原始量纲。方法如下：对于每个测试样本，取其最后一个时间步的其它特征（非污染值），将预测污染值与这些实际特征拼接成完整向量，然后整体应用`scaler.inverse_transform`逆缩放【42†L204-L210】。最后提取逆缩放后的首列作为污染浓度的预测值`inv_y_pred`；真实值`inv_y_test`同理处理。这样可以保证预测和实际都被还原到原始单位以计算指标【42†L204-L210】。  

## 6. 实验结果  
- **损失曲线**：训练过程中记录了训练集和验证集的损失曲线。一般会观察到训练损失（MAE）随Epoch下降，验证损失趋于平稳或先下降后略微回升。早停起到了抑制验证损失震荡的作用（见图示）。  
- **性能指标**：对测试集，计算预测值与真实值之间的均方根误差（RMSE）。RMSE定义为$\sqrt{\frac{1}{N}\sum (y_i-\hat y_i)^2}$，能够直观衡量预测精度。实际运行中，我们得到如`Test RMSE: X.XX`（具体数值根据数据而定）。  
- **预测效果**：可视化全测试集和局部（如前200个样本）的预测结果曲线，对比真实值与预测值。大致上，模型能够捕捉污染的主要波动趋势，但在某些尖峰区域误差较大。这与数据本身的噪声和LSTM的预测能力有关。  
- **误差分布**：将预测误差$\epsilon_i=y_i-\hat y_i$绘制直方图，可观察误差分布大致呈对称分布，但可能有较长尾巴，表明有少数点偏差较大。误差的统计分析可指示模型的系统性偏差及改进方向。  

## 7. 结果讨论与改进建议  
1. **模型选择与超参数**：本次实验采用了堆叠双层LSTM结构和Dropout正则化，这与常用实践一致【42†L159-L168】。可尝试调整单层/多层、单元数或Dropout比例，以及更改损失函数（如MSE）比较效果差异。  
2. **数据特征**：当前使用了所有气象变量，但原始数据还包含年、月、日、小时等时间属性未直接利用。后续可考虑引入周期性时间特征（如一周周期、季节周期等）或采用时间嵌入（time embedding）提高模型对长周期波动的感知。  
3. **多步预测**：本实验仅预测下一小时污染浓度，未来可扩展为多步预测（multi-step forecasting），预测未来多小时的污染水平，以评估模型在更长时域上的性能。  
4. **更多正则化**：除了Dropout，还可以尝试L2正则、批标准化等技术。增加EarlyStopping监控指标，比如同时监控验证RMSE而非仅MAE，有助于得到均方误差更低的模型。  
5. **数据增强**：考虑对数据进行平滑、去噪或引入外部数据（如节假日信息、交通量数据等）以丰富特征，可能提高预测精度。  
6. **模型集成**：可将LSTM模型与传统时间序列模型（如ARIMA）或其他神经网络（如GRU、变压器模型）进行集成，以期提升稳定性和准确性。  

## 8. 小结  
本实验验证了利用堆叠LSTM对多变量空气质量时间序列进行建模与预测的可行性。通过数据预处理（标签编码、归一化）、序列化处理和网络训练，模型能够在一定程度上准确预测未来污染水平。结果表明，该方法符合相关文献中对多变量时序预测的通用流程【42†L125-L134】【42†L159-L168】。基于实验观察，我们给出了若干改进方向以进一步提升模型性能。未来工作可通过更丰富的数据特征和先进的深度学习架构来持续优化预测效果。
